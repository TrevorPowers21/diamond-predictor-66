### Auto-fire (live on prod)
- Trigger: `AFTER INSERT ON customer_teams` fires `fn_customer_teams_autofire_precompute()`
- Inserts TWO rows in `precompute_jobs` (one `hitters_d1`, one `pitchers_d1`)
- Fires HTTP POST to Edge Function for each job
- Edge Function dispatches to `runHitterPrecompute` or `runPitcherPrecompute` based on `job.scope`
- Vault secrets: `precompute_edge_function_url`, `precompute_service_role_key`

### Locked rows
`protect_locked_predictions` trigger reverts rate columns when `locked = true`. Originally meant to protect coach-saved values from batch recomputes. **ALL precompute scripts must write `locked: false`** so subsequent runs can refresh. Don't manually lock rows from profile clicks (the bug we hit on 2026-05-23 — fixed by making profile dropdowns session-only).
